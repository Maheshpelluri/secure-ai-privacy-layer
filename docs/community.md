# Community Discussions

GitHub Discussions should be used for open-ended conversation that does not fit
a discrete issue.

## Recommended Categories

- General: repository-wide announcements and broad community questions.
- Architecture: conceptual design, system boundaries, and tradeoff discussions.
- Research: prior art, hypotheses, validation ideas, and open questions.
- Implementation: future prototype mechanics once coding begins.
- Ideas: early proposals that are not yet ready for a formal RFC or issue.
- Show and Tell: related work, experiments, or external examples.
- Q&A: direct questions about how the repository is organized or how to
  contribute.

## Usage Guidance

- Use issues when a conversation has a concrete action item or repository
  change.
- Use discussions when the goal is exploration, debate, or community knowledge
  sharing.
- Promote a discussion to an RFC or issue once the topic has a clear decision
  path.
