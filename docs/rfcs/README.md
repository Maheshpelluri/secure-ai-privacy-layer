# RFCs

Future architectural changes should be proposed through the RFC process before they are adopted.

## Purpose

RFCs provide a structured way to propose, discuss, and record significant architectural or governance changes.

## When to Write an RFC

- The change affects architecture, scope, or boundary definitions.
- The proposal needs discussion across multiple options.
- The repository would benefit from a durable decision record.

## Process

1. Start with `RFC_TEMPLATE.md`.
2. Describe the problem, proposal, tradeoffs, and open questions.
3. Review the RFC in public discussion or an issue.
4. Record the final disposition after the review cycle.
