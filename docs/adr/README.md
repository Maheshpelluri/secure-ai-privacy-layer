# Architecture Decision Records

This directory records future architecture decisions in a lightweight,
reviewable form.

## Purpose

ADR files capture why a decision was made, what alternatives were considered,
and what the consequences are.

## When to Use an ADR

- A proposal changes the architecture, trust boundaries, or system
  responsibilities.
- Multiple options exist and the tradeoffs should be preserved.
- The decision is important enough that future contributors will need its
  rationale.

## Naming Convention

- Use zero-padded numeric prefixes for chronological ordering.
- Keep titles short and descriptive.

## Template Usage

Copy `000-template.md` when creating a new ADR and replace the placeholder
sections with the decision details.
